# Contributing

## Branch Naming

- Frontend work: `frontend/<username>-task`
- Backend work: `backend/<username>-task`

Use your own GitHub username when possible. If your username is not yet confirmed, use a readable name such as `frontend/kandere-ndl-task`.

## Commit Rules

- Commit only your own work.
- Use your own GitHub account and Git identity.
- Do not commit on behalf of another student.
- Use clear commit messages, for example: `Add dashboard layout` or `Implement validation for inspection records`.

## Pull Requests

- Push your branch to GitHub.
- Open a pull request into `main`.
- Describe what you changed and how it was tested.
- Wait for team leader review before merge.

## Assigned Folders

- Frontend students should primarily work in `frontend/`.
- Backend students should primarily work in `backend/`.
- Shared documentation and schemas belong in `shared/`.
- Planning and review documents belong in `project-management/`.

## Secrets Rule

Never commit passwords, credentials, tokens, private keys, `.env` files, or raw account/login files. If a secret is accidentally added, notify the team leader immediately before pushing.

## Code Quality

Keep changes focused on your assigned task. Use readable names, add useful validation, document setup steps, and test the feature before opening a pull request.
